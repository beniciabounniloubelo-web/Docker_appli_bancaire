=> test du backend avec Postman ou curl pour verifier que les requetes retournent bien les donnees
requete dans le terminal apres:
curl http://localhost:8000/comptes/1

pour le docker compose:
backend:
  build: ./backend
  ports:
    - "8000:8000"
  depends_on:
    - postgres

pour le backend/dockerfile:
FROM python:3.11
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

pour lancer dans le terminal:
docker-compose up --build

pour le termianl pour tester:
curl http://localhost:8000/comptes/1

Notes importantes:
Deux choses importantes à savoir : le %s dans les requêtes SQL c'est pour éviter les injections SQL (ne jamais coller les variables directement dans la string), et le conn.commit() est obligatoire après chaque INSERT ou UPDATE pour que les changements soient bien sauvegardés en base.
